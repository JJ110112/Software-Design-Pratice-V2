## 一、系統概觀

本系統是針對「技術士技能檢定電腦軟體設計丙級」術科考試所設計的互動式練習平台，涵蓋 VB.NET 程式語言的基本指令操作與應用程式設計。系統採用由淺入深的五階段學習路徑，搭配 16 種練習模式、模擬考功能，以及完整的學習進度追蹤系統。

### 技術架構

- **前端**：純 HTML/CSS/JavaScript（無框架）
- **後端**：Firebase Firestore（雲端資料庫）
- **部署**：靜態網站（npx serve / GitHub Pages）
- **測試**：Playwright E2E 自動化測試

## 三、學習階段與練習流程

### 五階段由淺入深

| 階段 | 名稱 | 練習模式 | 顏色 | 星星上限 |
|------|------|---------|------|---------|
| 1 | 認識指令（基礎） | 連連看、記憶翻牌、中英選擇題 | 青色 | 171 |
| 2 | 讀懂一行程式碼 | 朗讀練習、一行翻譯、錯誤找找看 | 綠色 | 171 |
| 3 | 讀懂整段程式 | 排列重組、結果配對、注解填空 | 藍色 | 171 |
| 4 | 自己動手寫 | 打字-關鍵字、打字-單行、看中文寫程式、打字-完整 | 紫色 | 228 |
| 5 | 模擬考試情境 | 程式填空、獨立全程式撰寫、錯誤程式除錯 | 金色 | 171 |

**總星星數：912**

### 每個練習模式的關卡序列（19 關）

```
SETUP(T01) → Q1(T01) → Q1(T02) → Q1(T03)
           → Q2(T01) → Q2(T02) → Q2(T03)
           → Q3(T01) → Q3(T02) → Q3(T03)
           → Q4(T01) → Q4(T02) → Q4(T03)
           → Q5(T01) → Q5(T02) → Q5(T03)
           → 1060306(T01) → 1060307(T01) → 1060308(T01)
```

### 迴圈變體

| 代號 | 迴圈類型 | VB.NET 語法 |
|------|---------|------------|
| T01 | For...Next | `For i = 0 To n` |
| T02 | Do While...Loop | `Do While 條件 ... Loop` |
| T03 | Do...Loop While | `Do ... Loop While 條件` |

### 階段銜接流程

```
第一階段最後（中英選擇題）
  → 第二階段（程式碼朗讀練習）
  → 第三階段（程式碼排列重組）
```

## 五、星星計算規則

### 各練習通用

| 條件 | 星星 |
|------|------|
| 完美通過（0 提示 / 0 錯誤） | 3 星 |
| 使用 1~3 次提示或少量錯誤 | 2 星 |
| 使用 >3 次提示或大量錯誤 | 1 星 |
| 作弊偵測（WPM > 120 或 < 5 秒完成） | 0 星（不儲存） |

### 防作弊機制

- 所有打字練習頁面**禁止貼上**（`paste` 事件被攔截）
- 打字練習 WPM > 120 → 0 星 + 紅字警告
- 獨立全程式撰寫 < 5 秒完成 → 0 星

## 六、資料同步機制

### 架構

```
瀏覽器 localStorage (快取)
    ↕ 樂觀更新
Firebase Firestore (雲端)
```

### 儲存流程（saveScore）

```
1. 先寫入本地快取（cacheAppend）→ 星星立即可見
2. 嘗試寫入 Firestore（addDoc）
3. 如果 Firestore 失敗 → 存入 local_scores 備份
```

### 讀取流程（getScoresForUser）

```
1. 檢查快取是否有效（TTL 2 小時）
2. 有效 → 直接返回快取 + 合併 local_scores
3. 過期 → 從 Firestore 重撈 → 建立新快取 → 合併 local_scores
```

### 同步觸發時機

| 時機 | 動作 |
|------|------|
| 登入 | 嘗試上傳離線成績（syncOnLogin） |
| 登出 | 清除該使用者快取（syncOnLogout） |
| 進入 map 頁面 | 節流背景同步，10 分鐘冷卻（syncOnMapLoad） |
| 手動 | Console 執行 `window.syncOfflineScores('使用者名')` |

### 快取 TTL

| 快取類型 | TTL |
|---------|-----|
| 個人成績 (fb_cache_使用者) | 2 小時 |
| 排行榜 (fb_cache_overall_ranking) | 4 小時 |
| Dashboard (fb_cache_dashboard_teacher) | 4 小時 |
| 地圖同步冷卻 (last_sync_使用者) | 10 分鐘 |

### Firestore 讀取量控管

| 措施 | 效果 |
|------|------|
| saveScore 不清除排行榜快取 | 避免頻繁重查 |
| syncOnMapLoad 10 分鐘節流 | 減少重複查詢 |
| 排行榜用 PASS 篩選查詢 | 減少不必要的資料 |
| Dashboard limit 500 | 控制每次讀取量 |

### 預估用量（30 人/天）

- 日均讀取：~15,000 次（Firestore 免費額度 50,000 次/天）

## 七、核心 JavaScript 模組

### api.js — 資料層

| 函數 | 用途 |
|------|------|
| `saveScore()` | 儲存過關成績（樂觀更新 + Firestore） |
| `getScoresForUser()` | 取得個人所有成績（快取優先） |
| `getUserStarStats()` | 計算個人星星統計（分階段、分模式） |
| `getOverallRanking()` | 綜合排行榜 |
| `getAllScoresForDashboard()` | 教師儀表板資料 |
| `syncOnLogin/Logout/MapLoad()` | 同步控制 |
| `syncOfflineScores()` | 離線成績重試上傳 |

### quiz_data.js — 題庫

- `QUIZ_DATA` 物件包含所有題目定義
- `generateSteps(qID, tID)` 根據題號和迴圈變體產生完整程式碼步驟
- `injectDynamicUserSetup()` 自動將登入使用者的姓名、座號、日期注入 SETUP 題目

### nextlevel.js — 關卡銜接

- 監聽通關 overlay 的 `.show` class 變化
- 根據當前頁面檔名和 q/t 參數決定下一關
- 處理跨階段的練習銜接

### game-toolbar.js — 工具列

- 自動注入到所有練習頁面
- 提供首頁、地圖、音效開關、重新開始按鈕
- 攔截 Web Audio API 和 Speech API 實現全域靜音
- 打字練習的 sub 參數會對應正確的 map mode

### users.js — 使用者管理

- `CLASS_ROSTER` 班級名單（資訊二、電子二）
- `getCurrentUser()` / `loginUser()` / `logoutUser()` 管理登入狀態
- `enforceLogin()` 全域登入防護（非首頁自動跳轉）

## 十一、常見問題排查

### 星星數不一致（不同瀏覽器）

**原因**：成績存在 localStorage 快取中，不同瀏覽器有獨立的 localStorage。

**解決**：
1. 檢查 Firestore 安全規則是否允許寫入
2. Console 執行 `JSON.parse(localStorage.getItem('local_scores') || '[]').length` 查看未上傳筆數
3. 執行 `window.syncOfflineScores()` 手動補傳
4. 清除快取：`localStorage.removeItem('fb_cache_使用者名')`

### 頁面內容被導覽列遮住

`common.css` 已設定 `body { padding-top: 56px !important; }` 來避免。如果仍有問題，可能是瀏覽器快取舊版 CSS，嘗試 Ctrl+Shift+R 強制重整。

### Firestore 讀取量過高

1. 檢查是否有頁面頻繁刷新
2. 確認 `saveScore` 不會清除排行榜快取
3. 確認 `syncOnMapLoad` 有 10 分鐘冷卻

### 打字練習輸入卡住

- 按「清除」按鈕重新開始當前題目
- 完整模式可用「清除一行」刪除最後一行
- 焦點丟失時系統會自動 100ms 後重新聚焦
